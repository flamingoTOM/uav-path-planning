## 项目概述

本项目实现了一个基于 **Q-learning** 算法的强化学习路径规划系统，目标是让智能体学会在 6×6 网格世界中从起点（左下角）导航到终点（右上角）。

### 问题定义
- **环境**: 6×6 的二维网格
- **起点**: (5, 0) - 左下角
- **终点**: (0, 5) - 右上角
- **动作**: 上、下、左、右四个方向移动
- **目标**: 找到从起点到终点的最短路径

## 强化学习基础原理

### 什么是强化学习？

强化学习（Reinforcement Learning, RL）是机器学习的一个分支，它通过 **试错** 的方式让智能体（Agent）学习如何在环境（Environment）中采取最优行动。

### 核心概念

#### 1. **智能体（Agent）**
执行动作并从经验中学习的实体。在我们的例子中，智能体就是在网格中移动的"玩家"。

#### 2. **环境（Environment）**
智能体所处的外部世界。在我们的例子中，环境就是 6×6 的网格世界。

#### 3. **状态（State）**
描述环境在某一时刻的完整信息。在我们的例子中，状态是智能体在网格中的位置，如 (2, 3)。

#### 4. **动作（Action）**
智能体可以采取的行为。在我们的例子中，动作包括：上、下、左、右。

#### 5. **奖励（Reward）**
环境对智能体动作的即时反馈，用数值表示。奖励函数设计：
- 到达目标：+100（高奖励）
- 撞墙（出界）：-1（小惩罚）
- 正常移动：-1（移动成本）

#### 6. **策略（Policy）**
智能体的行为准则，定义了在每个状态下应该采取什么动作。用 π(s) 表示。

#### 7. **价值函数（Value Function）**
评估某个状态或状态-动作对的"好坏"。

### 强化学习的工作流程

```
    ┌──────────┐
    │  智能体   │
    │ (Agent)  │
    └────┬─────┘
         │
         │ 选择动作 (Action)
         ↓
    ┌────────────┐
    │    环境     │
    │(Environment)│
    └────┬───────┘
         │
         │ 返回奖励 (Reward) 和新状态 (Next State)
         ↓
    ┌──────────┐
    │  智能体   │
    │  学习更新 │
    └──────────┘
```

**循环过程**:
1. 智能体观察当前状态
2. 根据策略选择一个动作
3. 环境返回新的状态和奖励
4. 智能体更新自己的知识（Q表）
5. 重复上述过程

## Q-learning 算法详解

### 什么是 Q-learning？

Q-learning 是一种 **无模型（Model-Free）** 的强化学习算法，它学习一个 **Q 函数**（也称为动作价值函数），用来估计在某个状态下采取某个动作的长期价值。

### Q 函数定义

$$Q(s, a)$$ 表示在状态 $$s$$ 下采取动作 $$a$$ 的期望累积奖励。

### Q 表（Q-Table）

Q 表是一个二维数组，用于存储所有状态-动作对的 Q 值：

```
        动作0  动作1  动作2  动作3
状态0   [0.5,  0.3,  0.2,  0.7]
状态1   [0.1,  0.9,  0.4,  0.2]
状态2   [0.6,  0.3,  0.8,  0.1]
...
```

- **行**: 代表不同的状态
- **列**: 代表不同的动作
- **值**: 代表该状态-动作对的 Q 值

### Q-learning 更新公式

这是 Q-learning 的核心公式：

$$Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]$$

**公式解读**:

1. **$$Q(s, a)$$**: 当前状态 $$s$$ 下采取动作 $$a$$ 的 Q 值
2. **$$\alpha$$** (alpha): **学习率**（0 < α ≤ 1）
   - 控制新信息的吸收速度
   - α = 0: 不学习任何新信息
   - α = 1: 完全忽略旧信息
   - 通常设置为 0.1 ~ 0.3

3. **$$r$$**: 执行动作 $$a$$ 后获得的 **即时奖励**

4. **$$\gamma$$** (gamma): **折扣因子**（0 ≤ γ ≤ 1）
   - 衡量未来奖励的重要性
   - γ = 0: 只关心即时奖励（短视）
   - γ = 1: 未来奖励和即时奖励同等重要（远视）
   - 通常设置为 0.9 ~ 0.99

5. **$$\max_{a'} Q(s', a')$$**: 在新状态 $$s'$$ 下所有可能动作的 **最大 Q 值**

6. **$$r + \gamma \max_{a'} Q(s', a')$$**: **TD 目标**（Temporal Difference Target）
   - 表示对未来累积奖励的估计

7. **$$r + \gamma \max_{a'} Q(s', a') - Q(s, a)$$**: **TD 误差**
   - 表示当前估计与实际情况的差距

### 更新公式的直观理解

```
新的Q值 = 旧的Q值 + 学习率 × (实际收益 - 预期收益)
```

- 如果实际收益 > 预期收益: Q 值增加（这个动作比预期好）
- 如果实际收益 < 预期收益: Q 值减少（这个动作比预期差）

### ε-贪婪策略（Epsilon-Greedy）

在训练过程中，智能体需要在 **探索（Exploration）** 和 **利用（Exploitation）** 之间平衡：

- **探索**: 尝试随机动作，发现潜在的更好策略
- **利用**: 选择当前已知的最优动作

**ε-贪婪策略**:
```
以概率 ε: 随机选择动作（探索）
以概率 1-ε: 选择 Q 值最大的动作（利用）
```

**ε 衰减**:
- 训练初期：ε = 1.0（完全探索，因为 Q 表还不准确）
- 训练中期：ε 逐渐减小（逐渐转向利用）
- 训练后期：ε ≈ 0.01（主要利用，偶尔探索）

## 代码实现详解

### 1. GridWorld 类 - 环境实现

```python
class GridWorld:
    def __init__(self, size=6):
        self.size = size
        self.start_state = (5, 0)  # 左下角
        self.goal_state = (0, 5)   # 右上角
        self.actions = {
            0: (-1, 0),  # 上
            1: (1, 0),   # 下
            2: (0, -1),  # 左
            3: (0, 1)    # 右
        }
```

**关键方法**:

#### `reset()` - 重置环境
```python
def reset(self):
    self.current_state = self.start_state
    return self.current_state
```
每个回合开始时调用，将智能体放回起点。

#### `step(action)` - 执行动作
```python
def step(self, action):
    # 1. 计算新位置
    row, col = self.current_state
    d_row, d_col = self.actions[action]
    new_row = max(0, min(self.size - 1, row + d_row))
    new_col = max(0, min(self.size - 1, col + d_col))

    # 2. 更新状态
    self.current_state = (new_row, new_col)

    # 3. 计算奖励
    if self.current_state == self.goal_state:
        reward = 100  # 到达目标
        done = True
    elif (new_row, new_col) == (row, col):
        reward = -1  # 撞墙
        done = False
    else:
        reward = -1  # 正常移动
        done = False

    return self.current_state, reward, done
```

**奖励函数设计思路**:
- **+100**: 到达目标 → 鼓励智能体达成目标
- **-1**: 每步移动成本 → 鼓励智能体找最短路径
- **-1**: 撞墙惩罚 → 让智能体学会避免无效动作

### 2. QLearningAgent 类 - 智能体实现

```python
class QLearningAgent:
    def __init__(self, n_states, n_actions, learning_rate=0.1,
                 discount_factor=0.95, epsilon=1.0):
        self.lr = learning_rate          # α = 0.1
        self.gamma = discount_factor     # γ = 0.95
        self.epsilon = epsilon           # ε = 1.0

        # 初始化 Q 表为全零
        self.q_table = np.zeros((n_states, n_actions))
```

**关键方法**:

#### `choose_action()` - 选择动作
```python
def choose_action(self, state_index, training=True):
    if training and np.random.random() < self.epsilon:
        # 探索：随机选择
        return np.random.randint(self.n_actions)
    else:
        # 利用：选择 Q 值最大的动作
        return np.argmax(self.q_table[state_index])
```

**逻辑解释**:
1. 生成 0~1 之间的随机数
2. 如果随机数 < ε: 随机选择动作（探索）
3. 否则: 选择 Q 表中该状态下 Q 值最大的动作（利用）

#### `update_q_table()` - 更新 Q 表
```python
def update_q_table(self, state, action, reward, next_state):
    current_q = self.q_table[state, action]
    max_next_q = np.max(self.q_table[next_state])

    # Q-learning 更新公式
    new_q = current_q + self.lr * (reward + self.gamma * max_next_q - current_q)
    self.q_table[state, action] = new_q
```

**公式对应**:
```
current_q           → Q(s, a)
reward              → r
max_next_q          → max Q(s', a')
self.lr             → α
self.gamma          → γ
```

#### `decay_epsilon()` - 衰减探索率
```python
def decay_epsilon(self):
    self.epsilon = max(self.epsilon_min, self.epsilon * self.epsilon_decay)
```

**衰减策略**:
- 每个回合后，ε 乘以衰减系数（如 0.995）
- 确保 ε 不会低于最小值（如 0.01）

### 3. train_agent() - 训练主循环

```python
def train_agent(env, agent, n_episodes=500, max_steps=100):
    for episode in range(n_episodes):
        # 1. 重置环境
        state = env.reset()
        state_index = env.get_state_index(state)

        for step in range(max_steps):
            # 2. 选择动作
            action = agent.choose_action(state_index, training=True)

            # 3. 执行动作
            next_state, reward, done = env.step(action)
            next_state_index = env.get_state_index(next_state)

            # 4. 更新 Q 表
            agent.update_q_table(state_index, action, reward, next_state_index)

            # 5. 更新状态
            state_index = next_state_index

            # 6. 检查是否到达终点
            if done:
                break

        # 7. 衰减探索率
        agent.decay_epsilon()
```

**训练流程**:
```
开始 → 重置环境 → 选择动作 → 执行动作 → 获得奖励 → 更新Q表 →
检查是否结束 → 如果未结束则继续，如果结束则进入下一回合 → 衰减ε
```

## 训练过程分析

### 训练阶段划分

#### 第一阶段：随机探索（回合 1-100）
- **ε 值**: 1.0 → 0.6
- **行为**: 智能体几乎完全随机移动
- **Q 表**: 逐渐积累初始经验
- **成功率**: 很低（< 20%）
- **现象**: 奖励波动大，步数不稳定

#### 第二阶段：学习改进（回合 100-300）
- **ε 值**: 0.6 → 0.2
- **行为**: 开始利用已学到的知识，但仍保持探索
- **Q 表**: 快速更新，逐渐收敛
- **成功率**: 快速提升（20% → 80%）
- **现象**: 奖励稳步上升，步数逐渐减少

#### 第三阶段：策略收敛（回合 300-500）
- **ε 值**: 0.2 → 0.01
- **行为**: 主要利用最优策略，偶尔探索
- **Q 表**: 接近最优值，变化很小
- **成功率**: 稳定在高水平（> 95%）
- **现象**: 奖励和步数趋于稳定

### Q 表的演化

**初始状态（回合 0）**:
```
所有 Q 值 = 0
智能体不知道哪个动作更好
```

**中期状态（回合 200）**:
```
靠近目标的状态 Q 值较高
远离目标的状态 Q 值较低
开始形成正确的方向指引
```

**最终状态（回合 500）**:
```
Q 值形成从起点到终点的"势能场"
每个状态都有明确的最优动作
智能体可以稳定找到最短路径
```

## 参数调优指南

### 1. 学习率（α, learning_rate）

**作用**: 控制新信息的权重

| α 值 | 效果 | 适用场景 |
|------|------|---------|
| 0.01-0.05 | 学习缓慢，需要更多回合 | 环境复杂，需要细致学习 |
| **0.1-0.3** | **平衡，推荐** | **大多数场景** |
| 0.5-1.0 | 学习快，但可能不稳定 | 简单环境，快速原型 |

**调试建议**:
- 如果 Q 值波动大 → 降低 α
- 如果收敛太慢 → 提高 α

### 2. 折扣因子（γ, discount_factor）

**作用**: 衡量未来奖励的重要性

| γ 值 | 效果 | 特点 |
|------|------|-----|
| 0.5-0.7 | 短视，只关注近期奖励 | 可能学到次优策略 |
| **0.9-0.95** | **平衡，推荐** | **长短期兼顾** |
| 0.99-1.0 | 远视，重视长期收益 | 收敛慢，但策略更优 |

**调试建议**:
- 如果智能体太"急功近利" → 提高 γ
- 如果训练太慢 → 降低 γ

### 3. 探索率及其衰减（ε, epsilon）

**初始值（epsilon）**:
- 推荐：1.0（完全探索）
- 快速测试：0.5

**衰减系数（epsilon_decay）**:
| 值 | 衰减速度 | 回合数达到 0.01 |
|----|---------|----------------|
| 0.99 | 慢 | ~460 回合 |
| **0.995** | **中等，推荐** | **~920 回合** |
| 0.999 | 慢 | ~4600 回合 |

**最小值（epsilon_min）**:
- 推荐：0.01（保持 1% 的探索）
- 不推荐设为 0（会完全停止探索）

### 4. 训练回合数（n_episodes）

**建议**:
- 简单环境（如本例）：300-500 回合
- 中等复杂度：1000-2000 回合
- 复杂环境：5000+ 回合

**判断标准**:
- 成功率稳定在 100%
- Q 值不再明显变化
- 步数稳定在最优值

### 5. 最大步数（max_steps）

**作用**: 防止无限循环

**建议值**:
- 理论最短路径的 2-3 倍
- 本例中：最短路径 10 步 → 设置 30-50 步
- 默认 100 步较为安全

## 常见问题与解决方案

### Q1: 训练后成功率不到 100%
**可能原因**:
1. 训练回合数不足
2. 探索率衰减太快
3. 学习率设置不当

**解决方案**:
- 增加训练回合数到 1000
- 降低 epsilon_decay 到 0.999
- 调整学习率到 0.1-0.2

### Q2: 找到的路径不是最短路径
**可能原因**:
1. 奖励函数设计不合理
2. 折扣因子太低
3. 陷入局部最优

**解决方案**:
- 增加到达目标的奖励（如 +200）
- 提高折扣因子到 0.99
- 保持足够的探索（epsilon_min = 0.05）

### Q3: Q 值爆炸或变成 NaN
**可能原因**:
1. 学习率太高
2. 奖励值太大

**解决方案**:
- 降低学习率到 0.01-0.05
- 调整奖励范围在 [-10, 100] 内

### Q4: 训练过程不稳定，奖励波动大
**可能原因**:
1. 学习率太高
2. 探索率衰减太快

**解决方案**:
- 降低学习率
- 使用更平滑的衰减策略

- **SARSA**: On-policy 算法，更稳定但可能不如 Q-learning 优化
- **Deep Q-Network (DQN)**: 使用神经网络代替 Q 表，适合大规模状态空间
- **Policy Gradient**: 直接学习策略函数

### 3. 动态环境
让终点位置随机变化，训练一个更通用的智能体。

### 4. 多智能体协同
多个智能体同时在网格中移动，避免碰撞，协同到达目标。
